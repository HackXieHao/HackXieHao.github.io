3 SpringMVC自动配置

详情访问Spring Boot官网关于The ‘Spring Web MVC framework’

3.1 自动配置SpringMVC的组件详解（对照官网）

  1. Inclusion of ContentNegotiatingViewResolver and BeanNameViewResolver beans.

    • 自动配置了ViewResolver（视图解析器：根据方法的返回值得到视图对象（View），视图对象决定如何渲染（如转发或重定向））
    • ContentNegotiatingViewResolver：组合所有的视图解析器的

    定制ViewResolver：我们可以自己给容器中添加一个视图解析器，自动的将其组合进来。

  2. Support for serving static resources, including support for WebJars (see below) ：静态资源文件夹路径，webjars

  3. Static index.html support. ：静态首页访问

  4. Custom Favicon support (see below). ：图标favicon.ico

  5. Automatic registration of Converter, GenericConverter, Formatter beans.

    • Converter：转换器； public String hello(User user)：类型转换使用Converter
    • Formatter ：格式化器； 2018.11.16 —– > Date；
  1. Support for HttpMessageConverters (see below).：SpringMVC用来转换Http请求和响应的，User—Json
  2. Automatic registration of MessageCodesResolver (see below). ：定义错误代码生成规则
  3. Automatic use of a ConfigurableWebBindingInitializer bean (see below).：初始化WebDataBinder，请求数据转换为JavaBean

4 总结：修改Spring Boot默认配置的方法

  1. SpringBoot在自动配置很多组件的时候，先看容器中有没有用户配置的（@Bean、@Component）：

    如果有，就用用户配置的；

    如果没有，才自动配置；

    如果有些组件可以有多个（如ViewResolver），则将用户配置的和默认配置组合起来

  2. 在SpringBoot中会有非常多的xxxConfigurer帮助我们进行扩展配置

  3. 在SpringBoot中会有很多的xxxCustomizer帮助我们进行定制配置
